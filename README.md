Проект посвящён классификации символов алфавита американского жестового языка (ASL) по изображению. Для проекта используется датасет ASL Alphabet (`https://www.kaggle.com/datasets/grassknoted/asl-alphabet`). На данный момент реализован бейзлайн.

Вход системы $-$ изображение, на котором показан один символ алфавита ASL, специальный символ (SPACE, DELETE) или не показывается ничего. Выход системы $-$ метка одного из $29$ классов и число, демонстрирующее уверенность модели в ответе. Подробная инженерная постановка задачи описана в `PROJECT_PASSPORT.md`.

Структура проекта:

`configs/baseline.yaml`: конфигурация бейзлайна для воспроизводимости полученного результата

`configs/baseline_v2.yaml`: конфигурация бейзлайна для воспроизводимости проверки чувствительности результатов к протоколу

`data/raw_data`: фотографии из ASL Alphabet, разделённые по папкам, названия которых соответствуют корректным классам

`reports/baseline_analysis.md`: анализ результатов, полученных после обучения бейзлайна

`reports/data_audit.md`: аудит данных

`runs/baseline_v1/`: артефакты, полученные в результате запуска бейзлайна

`runs/baseline_v2/`: артефакты, полученные в результате запуска бейзлайна для проверки чувствительности результатов к протоколу

`splits/v1.json`: фиксированное разбиение данных на обучающую, тестовую и валидационную выборки для бейзлайна

`splits/v2.json`: фиксированное разбиение данных на обучающую, тестовую и валидационную выборки для проверки чувствительности результатов к протоколу

`src/analyze_data.py`: код для анализа различных статистических характеристик изображений в датасете

`src/create_split.py`: код для разбиения данных на выборки для бейзлайна

`src/create_split_v2.py`: код для разбиения данных на выборки для бейзлайна для проверки чувствительности результатов к протоколу

`src/dataset.py`: создание датасета по фиксированному разбиению данных

`src/evaluate.py`: код для вычисления метрик качества

`src/seed.py`: код для фиксирования всех сидов для воспроизводимости результатов

`src/train.py`: код для обучения и валидации бейзлайна

`PROJECT_PASSPORT.md`: описание подробной инженерной постановки задачи

В качестве бейзлайна выбрана модель ResNet18, в качестве метрик используются accuracy и macro-f1. Для воспроизведения результатов требуется:

$1.$ Запустить ```python src/create_split.py```

$2.$ Запустить ```python src/train.py```, предварительно указав `configuration_path = 'configs/baseline.yaml'`

$3.$ После этого в папке `runs/baseline_v1/` будут сохранены `best.pt` $-$ чекпоинт лучшей модели, а также `result.json` $-$ итоговые метрики.
